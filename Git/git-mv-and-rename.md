# Git Move (mv) සහ Rename

Git වලදී File එකක නම වෙනස් කිරීම හෝ තිබෙන තැන මාරු කිරීම නිවැරදිව සිදු කරන්නේ කෙසේද?

### 1. File එකක නම වෙනස් කිරීම (Rename)
සාමාන්‍ය `rename` එකක් වෙනුවට පහත command එක පාවිච්චි කරන්න:
`git mv old-name.md new-name.md`

### 2. File එකක් තවත් තැනකට මාරු කිරීම (Move)
`git mv file.txt ./new-folder/file.txt`

### ඇයි `git mv` වැදගත් වෙන්නේ?
- **History Preservation:** අදාළ File එකේ අතීතයේ කරපු සියලුම වෙනස්කම් (Commits) අලුත් නමටත් හිමිවේ.
- **Auto-Staging:** ඔබ නම වෙනස් කළ සැණින් Git විසින් එය `Staged` කරනු ලබයි. ඔබට අමුතුවෙන් `git add` කිරීමට අවශ්‍ය නොවේ.

### අමතක වී සාමාන්‍ය විදිහට නම වෙනස් කළොත්?
බය වෙන්න එපා. ඔබ සාමාන්‍ය විදිහට නම වෙනස් කළත්, `git add .` කළ විට Git බොහෝ විට එය "Rename" එකක් ලෙස ඉබේම හඳුනා ගනී (Similarity index එක අනුව). නමුත් ආරක්ෂිතම ක්‍රමය `git mv` පාවිච්චි කිරීමයි.
